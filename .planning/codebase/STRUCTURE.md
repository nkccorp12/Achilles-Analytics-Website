## Directory Purposes

**`public/`:**
- Purpose: Unmodified static assets served at root path during development and production
- Contains: Images (PNG, JPG, WebP), videos (MP4), favicons
- Accessed via: `<img src="/map.webp" />` (direct paths)
- Generated: No
- Committed: Yes

**`src/`:**
- Purpose: All source code (JSX, CSS, component logic)
- Contains: React components, pages, styling, entry point
- Key files: `main.jsx` (app bootstrap), `App.jsx` (routing), `index.css` (theme)
- Generated: No
- Committed: Yes

**`src/components/`:**
- Purpose: Reusable interactive UI elements and visualizations
- Contains: Three component modules: CardSwap (animation carousel), DigitalTwinViz (3D graph), LaserFlow (shader effect)
- Key files:
  - `CardSwap.jsx` - Exported as default and named export `Card`
  - `DigitalTwinViz.jsx` - Default export hook-based component
  - `LaserFlow.jsx` - Default export hook-based component with GLSL shaders
- Naming: PascalCase for component names, componentName.jsx and componentName.css pairs
- Generated: No
- Committed: Yes

**`src/pages/`:**
- Purpose: Full-page section components composed into landing/secondary pages
- Contains: CoreEngine, UseCases, IntelStack, AICouncil, NeuPage
- Pattern: Each page is a self-contained vertical section with heading, content, visualization
- Naming: PageName.jsx and PageName.css pairs
- Generated: No
- Committed: Yes

**`src/variants/`:**
- Purpose: Design system implementations and layout patterns
- Contains: VariantGrid (tactical grid design), VariantGlass, VariantNeon (alternative aesthetic variants)
- VariantGrid exports: `ArchitectureSection`, `PhilosophySection`, default export
- Usage: VariantGrid is main landing page pattern; others are alternative designs
- Generated: No
- Committed: Yes

**`src/assets/`:**
- Purpose: Bundled images/media (only react.svg currently)
- Generated: No
- Committed: Yes

**`dist/`:**
- Purpose: Production build output
- Generated: Yes (by `npm run build`)
- Committed: No

**`.planning/codebase/`:**
- Purpose: Architecture documentation
- Generated: No (created manually)
- Committed: Yes

## Naming Conventions

**Files:**
- React Components: `PascalCase.jsx` (e.g., `CardSwap.jsx`, `CoreEngine.jsx`)
- Styling: `lowercase-with-dashes.css` or `match-component-name.css` (e.g., `CoreEngine.css` paired with `CoreEngine.jsx`)
- Config: `lowercase.js` or `lowercase.cjs` (e.g., `vite.config.js`, `.eslintrc.cjs`)

**Directories:**
- Functional grouping: lowercase (e.g., `components/`, `pages/`, `variants/`)

**CSS Classes:**
- Pattern: `vg-` prefix for "Variant Grid" namespace
- Structure: Block-Element-Modifier (BEM) style with double dash separators
  - Block: `vg-engine` (component name)
  - Element: `vg-engine__panel` (sub-element)
  - Modifier: `vg-engine__panel--impact` (variant)
- Example class hierarchy: `vg-engine__split` > `vg-engine__panel` > `vg-engine__panel-header` > `vg-engine__panel-icon`

**React Component Props:**
- camelCase: `cardDistance`, `verticalDistance`, `pauseOnHover`, `skewAmount`
- Event handlers: `onCardClick`, `onClose`
- Config objects: Passed inline or destructured

**Functions & Variables:**
- camelCase: `makeSlot()`, `placeNow()`, `hideNode()`, `resetAll()`
- Constants: UPPER_SNAKE_CASE: `INTEL_PRODUCTS`, `STRATEGIES`, `NODES`, `ARCH_PILLARS`

**Inline SVG Icons:**
- Pattern: PascalCase function components (e.g., `IconBolt()`, `IconShield()`, `IconWarehouse()`)
- Scope: Defined within each page/component file (no separate icon library)

## Where to Add New Code

**New Feature (UI Section):**
- Primary code: Create `src/pages/FeatureName.jsx` with paired `src/pages/FeatureName.css`
- Import in: `src/App.jsx` (add import and include in LandingPage composition or new route)
- Style template: Use `vg-` namespace prefix, follow BEM pattern
- Icon template: Define inline SVG functions within the file

**New Component (Reusable Widget):**
- Implementation: Create `src/components/ComponentName.jsx` with paired `src/components/ComponentName.css`
- Export: Default export for primary component, named exports for sub-components (e.g., `Card` in CardSwap)
- Props: Define destructuring pattern for config
- Hooks: Use React hooks (useEffect, useRef, useState, useMemo, useCallback) as needed
- Animation: Use GSAP or Three.js depending on complexity

**Utilities & Helpers:**
- Shared helpers: Create `src/utils/` directory and `src/utils/helperName.js`
- Import pattern: `import { helper } from '../utils/helperName'`

**Styling:**
- Global CSS: Add to `src/index.css` for variables or shared resets
- Component CSS: Collocate with component file, use `vg-` namespace
- Responsive: Use CSS media queries; no Tailwind/utility framework detected
